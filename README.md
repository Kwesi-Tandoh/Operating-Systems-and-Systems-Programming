## Learning Goals

By the end of the course, students should be able to:

- **Explain the role of an operating system** and its place between users, applications, and hardware.
- **Describe major OS structures and architectures** such as monolithic, microkernel, layered, and hybrid designs.
- **Understand process management concepts** including process states, scheduling, threads, synchronization, and deadlocks.
- **Analyze memory management techniques** such as paging, segmentation, virtual memory, and page replacement.
- **Explain file systems and directory management** as core OS responsibilities.
- **Understand system calls, interrupts, and the boot process** as key mechanisms through which operating systems operate and interact with programs and devices.

## Core Course Themes

The course materials in this repository revolve around the following major areas:

- **OS Foundations**  
  Meaning, purpose, tasks, and evolution of operating systems.

- **OS Architecture**  
  Kernel, user space vs kernel space, monolithic systems, microkernels, layered and hybrid architectures.

- **Process and CPU Management**  
  Processes, process states, PCB, scheduling, scheduling algorithms, and context switching.

- **Concurrency**  
  Threads, multithreading, synchronization, critical sections, and deadlocks.

- **Memory Management**  
  Logical vs physical memory, paging, segmentation, virtual memory, and page replacement.

- **Storage and File Systems**  
  File management, directory structures, allocation methods, and access control.

- **System Interaction Mechanisms**  
  System calls, interrupts, and boot-time initialization.
